# Security Policy

## Supported Versions

This repository is a personal portfolio project. Only the **latest deployed version** is actively maintained.

| Version | Supported |
|----------|------------|
| Latest   | ✅ Yes |
| Older releases | ❌ No |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### How to report
- **Do NOT** open a public issue for security vulnerabilities.
- Email me directly at: onlyzekedotcom@gmail.com
- Or contact me via GitHub: https://github.com/defzeke

### What to include
Please provide:
- A detailed description of the vulnerability  
- Steps to reproduce  
- Potential impact  
- Any proof-of-concept code (if applicable)

---

## Response Timeline

I aim to:
- Acknowledge reports within **48 hours**
- Investigate and assess severity within **7 days**
- Patch and release a fix as soon as possible

---

## Security Best Practices

This project follows basic security practices such as:
- Dependency updates  
- Secure deployment via Vercel/GitHub Pages  
- No sensitive credentials stored in the repository  

---

## Disclaimer

This is a personal portfolio project and is not intended for production-critical use.  
Use at your own risk.
